# 1.1.3

[2 lakh marooned in Sunamganj flood | Prothom Alo](https://tse2.mm.bing.net/th/id/OIP.HAB2m-_8nRVA62ep-v-quAHaEn?pid=Api)

নিচের scenario-টা **actual current news claim না**; এটা আপনার research design explain করার জন্য realistic case simulation। ভোলা বাদ দিয়ে আমি নিচের area ধরছি:

# Case Area: টাঙ্গুয়ার হাওর, তাহিরপুর, সুনামগঞ্জ

**Tahirpur Upazila** সুনামগঞ্জ জেলার একটি flood/flash-flood-prone haor region। Banglapedia অনুযায়ী Tahirpur Upazila-এর আয়তন **315.33 sq km**, অবস্থান প্রায় **25°01′–25°12′N** এবং **91°02′–91°19′E**। ([Banglapedia](https://en.banglapedia.org/index.php/Tahirpur_Upazila?utm_source=chatgpt.com))

এই upazila-এর ভিতরে **Tanguar Haor** আছে, যেটি প্রায় **100 sq km** area cover করে এবং haor-এর ভিতরে/আশেপাশে বহু গ্রাম আছে। Banglapedia অনুযায়ী Tanguar Haor-এর area প্রায় 100 sq km, যার মধ্যে 2,802.36 ha wetland। ([Banglapedia](https://en.banglapedia.org/index.php/Tanguar_Haor?utm_source=chatgpt.com))

এখন ধরে নেই:

> **আজ তাহিরপুরের টাঙ্গুয়ার হাওর এলাকায় flash flood হয়েছে। পানি দ্রুত বেড়েছে, রাস্তা ডুবে গেছে, মোবাইল network unstable/down, electricity নেই, কিছু গ্রাম দ্বীপের মতো isolated হয়ে গেছে।**
> 

আপনার system এই অবস্থায় কীভাবে start-to-end কাজ করবে সেটা নিচে দিলাম।

# 1. Disaster হওয়ার আগেই system বসানো থাকবে

এই system flood হওয়ার পরে install করলে হবে না। আগে থেকেই high-risk zone-এ install করা থাকবে।

## Target deployment zone

```
Area: Tanguar Haor flood-risk zone
Approx. coverage: 100 sq km
Command side: Tahirpur Upazila control point + Sunamganj rescue coordination
Disaster type: flash flood / haor flood
Primary communication failure: mobile network + road access
```

## Device deployment plan

এই 100 sq km area-এর জন্য research-level practical deployment:

| Device type | Quantity | কাজ |
| --- | --- | --- |
| Small SOS Device | 45–60 | মানুষ SOS পাঠাবে |
| Local Mesh Relay | 12–18 | haor village cluster থেকে message collect করবে |
| Elevated Backbone Relay | 6–8 | দূরের upazila/rescue gateway পর্যন্ত relay corridor বানাবে |
| Boat Mobile Relay | 4–6 | disconnected village থেকে message collect করবে |
| Backpack Relay | 3–5 | volunteer/manual mobile relay |
| Command Gateway | 2 | primary + backup control center |

Minimum pilot করলে:

```
10 SOS + 3 Local Relay + 3 Backbone Relay + 1 Boat Relay + 1 Gateway = 18 devices
```

Full research simulation করলে:

```
50 SOS + 15 Local Relay + 7 Backbone Relay + 5 Boat Relay + 2 Gateway = 79 devices
```

আপনার proposal-এ ESP32/SX1276 LoRa node, solar/battery, IP-rated enclosure, GPS, heartbeat/self-healing routing, TinyML/Agentic AI, এবং backpack/mobile node concept আছে। এই scenario-তে সেগুলোকে practical disaster deployment হিসেবে সাজানো হচ্ছে।

# 2. Device কোথায় বসবে

## 2.1 Small SOS Device

এই device হবে সবচেয়ে ছোট এবং user-facing।

```
Approx. size: 10 cm × 7 cm × 4 cm
Controller: ESP32 / STM32 / nRF52
Radio: SX1276 / SX1262 LoRa
Power: rechargeable battery + small solar
Input: SOS button + 4 option buttons
Output: LED + buzzer
Protection: IP67 enclosure
Location: pre-registered node location
```

এগুলো বসবে:

```
- হাওরের village cluster
- mosque
- school
- flood shelter
- boat ghat
- market
- raised homestead
- embankment-side settlement
```

প্রতিটি device-এর ID থাকবে:

```
SOS-TGH-001
SOS-TGH-002
SOS-TGH-003
...
```

প্রতিটি node আগে থেকেই database-এ mapped থাকবে:

```
Node ID: SOS-TGH-014
Area: Tanguar Haor, Tahirpur
Village/Cluster: East Haor Settlement
Nearest landmark: Mosque-2 / boat ghat
GPS: pre-registered
Risk level: High
```

এখানে fixed SOS node হলে GPS না থাকলেও চলবে, কারণ **Node ID থেকেই location জানা যাবে**।

---

## 2.2 Local Mesh Relay

Local relay SOS device থেকে message collect করে backbone relay-তে পাঠাবে।

```
SOS Device → Local Mesh Relay → Elevated Backbone Relay
```

Local relay বসবে:

```
- school roof
- mosque roof
- cyclone/flood shelter roof
- tall bamboo/steel pole
- union-level high point
```

Local relay সাধারণ SOS device-এর চেয়ে একটু বড় হবে:

```
Box size: 15–20 cm
Solar panel: 10–20 W
Battery: LiFePO4 / multiple 18650
Antenna: external waterproof antenna
Mounting height: 12–20 feet
```

---

## 2.3 Elevated Backbone Relay Corridor

এটাই rescue team পর্যন্ত message পৌঁছানোর আসল path।

যদি শুধু হাওরের ভিতরে mesh থাকে, তাহিরপুর/সুনামগঞ্জ rescue team জানবেই না। তাই আগে থেকেই relay corridor থাকবে।

Example route:

```
Tanguar Haor Village SOS
    ↓
Local Relay at village school/mosque
    ↓
Backbone Relay-1: Haor-side high pole
    ↓
Backbone Relay-2: Tahirpur-side elevated point
    ↓
Backbone Relay-3: Upazila approach point
    ↓
Primary Command Gateway: Tahirpur Upazila control point
    ↓
Backup / Forwarding: Sunamganj District rescue coordination
```

Safe design rule:

```
SOS device walking coverage: 300–700 m
Local relay range planning: 1–2 km
Backbone relay spacing: 5–8 km
Gateway: Upazila / police / fire service / rescue control point
```

LoRaWAN সাধারণত gateway-centric architecture; কিন্তু আপনার system-এ disaster-time peer-to-peer forwarding করার জন্য **custom LoRa mesh layer** থাকবে। তাই technically এটাকে বলা ভালো: **LoRaWAN-compatible self-healing LoRa mesh**। LoRaWAN-compatible gateway final integration হিসেবে থাকবে, আর mesh routing custom protocol দিয়ে হবে।

## 2.4 Boat Mobile Relay

হাওর এলাকায় boat relay সবচেয়ে গুরুত্বপূর্ণ। কারণ flood হলে road-based rescue slow বা impossible হতে পারে।

Boat relay থাকবে:

```
- rescue boat
- local volunteer boat
- fire service/rescue boat
- medical support boat
```

কাজ:

```
1. disconnected SOS node থেকে stored message collect করবে
2. nearby SOS node detect করবে
3. critical incident list দেখাবে
4. gateway range পেলে message upload করবে
5. rescue team-কে nearest emergency দেখাবে
```

## 2.5 Command Gateway

Primary gateway থাকবে তাহিরপুর upazila control point-এ। Backup gateway থাকবে আরেকটি safer location-এ, যেমন police station/rescue control/shelter command point।

Gateway hardware:

```
Raspberry Pi / mini PC
LoRa concentrator / multiple LoRa radios
High-gain antenna
Local dashboard
Offline map
Siren/buzzer
Battery/solar/IPS backup
Optional cellular/satellite uplink
```


# 14. ACK না এলে কী হবে

Retry policy:

```
Try 1: immediate
Try 2: after 30 sec
Try 3: after 1 min
Try 4: after 3 min
Then: every 5 min
```

Same `message_id` থাকবে, তাই duplicate incident তৈরি হবে না।

Status:

```
CREATED → SENT → WAITING_ACK → RETRYING → DELIVERED
```

# 22. Agentic AI কোথায় কাজ করছে

আপনার title অনুযায়ী system-এ AI কাজ করছে ৭ জায়গায়:
